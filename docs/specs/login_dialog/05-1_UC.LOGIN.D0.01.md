**Версия:** 2.0 (итоговая)  
**Дата:** 2026-06-04  
**Автор:** Солодюк В.Л.

## 1. Введение

### 1.1 Цель документа
Детально описать автоматический сценарий инициализации модуля при запуске: отображение графической формы (на базе CustomTkinter) и загрузка сохранённых параметров из JSON-файла с поддержкой «умного фокуса» и значений по умолчанию.

### 1.2 Область применения
Документ предназначен для разработчиков и тестировщиков при реализации графического модуля.

### 1.3 Источники требований
- Концепция создания продукта / фичи (v3.4)
- Требования заинтересованных сторон (v2.5)
- Пользовательские истории (v2.3)
- Спецификация требований (v2.8)
- Список и диаграмма вариантов использования (v5.6)

## 2. Табличное описание варианта использования

| Атрибут | Значение |
|---------|----------|
| **ID** | UC.LOGIN.D0.01 |
| **Название** | Загрузить параметры и отобразить форму ввода данных |
| **Связи** | Отсутствуют |
| **Домен** | Инициализация |
| **Описание** | Вызывающая система вызывает функцию модуля (в режиме библиотеки или subprocess). Модуль отображает форму с полями «IP-адрес/хост», «Порт», «Имя пользователя», «Пароль», «Идентификатор службы (SID/Service Name)» и кнопками «Ок», «Отмена». Затем модуль автоматически загружает сохранённые параметры из JSON-файла. При отсутствии или повреждении файла поля заполняются значениями по умолчанию (Порт = 1521, Идентификатор службы = ORCL, остальные пустые). Поле пароля всегда пустое. Применяется логика «умного фокуса». |
| **Главные действующие лица** | Вызывающая система (A-2) |
| **Вовлеченные действующие лица** | Отсутствуют |
| **Предусловия** | 1. Модуль подключён к вызывающей системе как пакет (библиотека) или запущен как отдельный процесс (subprocess).<br>2. Графическое окно готово к отображению. |
| **Постусловия** | 1. Графическое окно отображено на экране (F-1).<br>2. Поля IP-адрес/хост, порт, имя пользователя, идентификатор службы заполнены значениями из файла (или значениями по умолчанию, если файла нет).<br>3. Поле пароля всегда пустое.<br>4. Кнопка «Ок» неактивна (F-4).<br>5. Фокус ввода установлен на поле «Пароль» (если поле `ip` или `username` не пустое) или на поле «IP-адрес/хост» (при первом запуске/отсутствии данных). |

## 3. Основной поток

| Шаг | Актор | Действие и логика системы |
|-----|-------|---------------------------|
| 1 | Вызывающая система | Вызывает функцию (API) модуля для отображения окна ввода. |
| 2 | Модуль | Отображает графическое окно с 5 полями и кнопками на базе CustomTkinter (F-1). |
| 3 | Модуль | Определяет путь к файлу: `%LOCALAPPDATA%\alphameterqc\connection.json` (Windows) или `~/.config/alphameterqc/connection.json` (Linux). |
| 4 | Модуль | Пытается прочитать файл. |
| 5 | Модуль | **Если файл существует и содержит корректный JSON (F-13):**<br>— Извлекает значения полей `ip`, `port`, `username`, `service_name`.<br>— Если поле отсутствует в JSON — использует значение по умолчанию.<br>— Подставляет полученные значения в соответствующие поля формы (F-8).<br><br>**Если файл отсутствует (F-9):**<br>— Новый файл не создаётся.<br>— Поля получают значения по умолчанию.<br><br>**Если файл повреждён или содержит некорректный JSON/типы данных (F-9):**<br>— Повреждённый файл удаляется.<br>— Поля получают значения по умолчанию.<br><br>**Если отсутствуют права на чтение (F-10):**<br>— Продолжает работу без загрузки параметров.<br>— Поля получают значения по умолчанию.<br>— Не выводит сообщений об ошибке. |
| 6 | Модуль | **Умный фокус (NF-1):**<br>— Если поле `ip` или `username` не пустое (заполнено из файла или передано как default), фокус ввода устанавливается на поле «Пароль» (для ускорения ввода).<br>— Иначе (первый запуск, все поля пусты) — фокус устанавливается на поле «IP-адрес/хост». |

## 4. Значения по умолчанию

| Поле | Значение по умолчанию |
|------|-----------------------|
| IP-адрес / Хост | Пустая строка `""` |
| Порт | `1521` |
| Имя пользователя | Пустая строка `""` |
| Пароль | Пустая строка `""` (всегда) |
| Идентификатор службы (SID/Service Name) | `"ORCL"` |

---

## 5. Обработка ошибок

| Ситуация | Реакция модуля |
|----------|----------------|
| Файл отсутствует | Не создавать новый. Поля = значения по умолчанию. Без сообщений. |
| Файл повреждён / некорректный тип данных | Удалить файл. Поля = значения по умолчанию. Без сообщений. |
| Нет прав на чтение | Продолжить работу без загрузки. Поля = значения по умолчанию. Без уведомлений. |
| Необработанное исключение в UI | Не допускать зависаний (NF-2b). Окно остаётся отзывчивым. |

## 6. Диаграмма деятельности (PlantUML)

```plantuml
@startuml
start
:Вызвать функцию модуля;
:Отобразить окно (CustomTkinter, F-1);
:Определить путь к JSON-файлу (%LOCALAPPDATA% / ~/.config/);
if (Файл существует?) then (да)
  if (Файл корректен?) then (да)
    :Подставить ip, port, username, service_name в поля (F-8, F-13);
    if (ip или username не пустые?) then (да)
      :Установить фокус на поле "Пароль" (Умный фокус, NF-1);
    else (нет)
      :Установить фокус на поле "IP-адрес/хост";
    endif
  else (нет)
    :Удалить повреждённый файл (F-9);
    :Использовать значения по умолчанию (Порт=1521, SID=ORCL);
    :Установить фокус на поле "IP-адрес/хост";
  endif
else (нет)
  :Использовать значения по умолчанию (F-9);
  :Установить фокус на поле "IP-адрес/хост";
endif
stop
@enduml
```

## 7. Диаграмма последовательности (PlantUML)

```plantuml
@startuml
actor "Вызывающая система" as Caller
participant "Модуль" as Module
database "Файловая система" as FS

Caller -> Module: 1. Вызвать функцию модуля
activate Module
Module -> Module: 2. Отобразить окно (F-1)
Module -> FS: 3-4. Прочитать JSON
alt Файл существует и корректен
  FS --> Module: ip, port, username, service_name
  Module -> Module: Подставить в поля (F-8, F-13)
  Module -> Module: Установить фокус на "Пароль" (если ip/username не пустые) или "IP-адрес/хост" (NF-1)
else Файл повреждён / некорректные типы
  FS --> Module: Ошибка
  Module -> FS: Удалить файл (F-9)
  Module -> Module: Поля = значения по умолчанию (1521, ORCL)
  Module -> Module: Установить фокус на "IP-адрес/хост"
else Файл отсутствует / нет прав
  Module -> Module: Поля = значения по умолчанию (F-9, F-10)
  Module -> Module: Установить фокус на "IP-адрес/хост"
end
deactivate Module
@enduml
```

## 8. Сводка покрытия требований (F)

| F-ID | Описание | Покрытие |
|------|----------|----------|
| F-1 | Отображение окна с 5 полями и кнопками (CustomTkinter) | Шаг 2 |
