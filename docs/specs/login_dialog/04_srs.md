## 1. Введение

### 1.1 Цель документа
Формальное описание функциональных и нефункциональных требований к модулю ввода идентификационных данных для подключения к БД Oracle. Спецификация является основой для разработки, тестирования и приёмки.

### 1.2 Область применения
Разработка графического модуля для ввода, валидации, безопасной передачи и локального сохранения (кроме пароля) параметров подключения к БД Oracle.

### 1.3 Источники требований
- Концепция создания продукта / фичи (v3.4)
- Требования заинтересованных сторон (v2.5)
- Пользовательские истории (v2.3)
- Список и диаграмма вариантов использования (v5.6)

### 1.4 Ссылки
- Метод MoSCoW (Must, Should, Could, Won't)
- Правила проверки полей (раздел 1.6)

### 1.5 Допущения и ограничения
- Целевая СУБД — Oracle (без Oracle Client на машине пользователя).
- Среда — Windows (основная, упаковка в .exe через PyInstaller), с поддержкой Linux.
- Файл сохранения — в локальной защищённой области пользователя (профиль).
- Файл не защищён от ручного редактирования, может быть повреждён.
- Модуль не выполняет сетевых запросов к БД до нажатия «Ок».
- Модуль НЕ проверяет доступность БД и корректность учётных данных — это ответственность вызывающей системы.
- Ошибки файловой системы обрабатываются молча (без вывода сообщений пользователю).
- Режимы интеграции: Модуль поддерживает два режима работы для исключения конфликтов графических циклов (Event Loop):
  - Режим библиотеки (по умолчанию): Модуль импортируется как Python-пакет, функция вызывается напрямую.
  - Режим отдельного процесса (альтернативный): Модуль запускается как самостоятельный процесс (subprocess), обмен данными происходит через stdout (JSON).

### 1.6 Правила проверки корректности полей

| Поле | Правило | Уточнения |
|------|---------|-----------|
| IP-адрес / Хост | Валидный IPv4-адрес или DNS-имя хоста | IPv4: четыре числа 0–255, разделённые точками. DNS: строка после удаления пробелов по краям (`strip`) должна содержать хотя бы один буквенно-цифровой символ, не начинаться и не заканчиваться на '-' или '.', допустимые символы: латиница, цифры, '_', '-', '.', не более 255 символов. |
| Порт | Число 1–65535 | Только десятичные цифры. Ведущие нули допустимы при вводе, но сохраняются как integer. |
| Имя пользователя | Не пустое, ≤ 30 символов | Латинские буквы, цифры, '_', '-'. |
| Пароль | Не пустой (≥ 1 символа) | Любые символы (пробелы, спецсимволы). Проверяется только непустота. |
| Идентификатор службы (SID/Service Name) | ≤ 30 символов | Допустимые символы: латиница, цифры, '_', '.', '-'. Поле опциональное (может быть пустым). |

### 1.7 Методы измерения производительности (для тестирования)

| Требование | Способ измерения | Критерий |
|------------|------------------|----------|
| NF-2a | Инструментальный замер обработчика события | ≤ 0,1 с (99% случаев) |
| NF-2b | Ручное тестирование с секундомером | Нет зависаний интерфейса длительностью более 0,5 секунды. Интерфейс должен оставаться отзывчивым при любых действиях пользователя и при любых состояниях системы. |
| NF-4 | Инструментальный замер от запуска до отрисовки окна | ≤ 0,5 с |

## 2. Функциональные требования

### Блок 1. Ввод, валидация и работа с файлом сохранения

| ID | Версия | Приоритет | Описание |
|----|--------|-----------|----------|
| F-1 | 2.8 | Must | Отображать окно с полями «IP-адрес/хост», «Порт», «Имя пользователя», «Пароль», «Идентификатор службы (SID/Service Name)» и кнопками «Ок», «Отмена». Подписи — на русском языке. Графический интерфейс реализуется с использованием библиотеки CustomTkinter. |
| F-2 | 2.8 | Must | Проверка корректности полей — в реальном времени при изменении содержимого (согласно правилам раздела 1.6). |
| F-3 | 2.8 | Must | Визуально подсвечивать поле, не соответствующее правилам (раздел 1.6). |
| F-4 | 2.8 | Must | Кнопка «Ок» неактивна, пока любое обязательное поле пустое или не соответствует правилам. Кнопка «Отмена» всегда активна. Если кнопка «Ок» неактивна, нажатие клавиши Enter игнорируется, окно не закрывается. |
| F-5 | 2.8 | Must | При нажатии «Ок» (или Enter при валидной форме) возвращать вызывающей системе структуру `{ip, port, username, password, service_name}`. |
| F-6 | 2.8 | Must | При нажатии «Отмена», закрытии окна через «X» или Esc возвращать сигнал отмены (`None` / `Cancel`). |
| F-7 | 2.8 | Must | При успешном завершении сеанса («Ок») сохранять в локальный файл IP-адрес/хост, порт, имя пользователя и идентификатор службы. Пароль не сохраняется. Для обеспечения атомарности записи на всех ОС, используется стандартная функция Python `os.replace(src, dst)`, которая гарантирует атомарную замену на POSIX-системах и корректно выполняет замену существующего файла на Windows, не требуя использования сторонних библиотек (ctypes). Временный файл (`.tmp`) создаётся строго в той же директории, что и целевой файл. |
| F-8 | 2.8 | Must | При запуске подставлять в поля IP-адрес/хост, порт, имя пользователя и идентификатор службы из сохранённого файла (согласно спецификации F-13). Поле пароля — всегда пустое. |
| F-9 | 2.8 | Must | При отсутствии файла сохранения или его повреждении — новый файл не создаётся, повреждённый файл удаляется. Поля заполняются значениями по умолчанию: `port` = 1521, `service_name` = "ORCL", остальные поля пустые. Сообщения об ошибке не выводятся. |
| F-10 | 2.8 | Should | При отсутствии прав на запись — продолжать работу, не сохраняя изменения и не выводя сообщений об ошибке. |
| F-11 | 2.8 | Must | Маскировать вводимый пароль символами-заменителями (например, «•»). |
| F-13 | 2.8 | Must | Формат файла: JSON (UTF-8), структура: `{"ip": "...", "port": 1521, "username": "...", "service_name": "..."}`. Пароль отсутствует. При чтении: игнорировать лишние поля. Для отсутствующих полей — значения по умолчанию (пустая строка для `ip`/`username`, 1521 для `port`, "ORCL" для `service_name`). Поле `port` всегда сериализуется как целое число (integer). При десериализации модуль должен корректно обрабатывать приведение строки к int, если файл был отредактирован вручную. Путь к файлу: `%LOCALAPPDATA%\alphameterqc\connection.json` (Windows) или `~/.config/alphameterqc/connection.json` (Linux). |

### Блок 2. Интеграция

| ID | Версия | Приоритет | Описание |
|----|--------|-----------|----------|
| F-12 | 2.8 | Must | Предоставлять задокументированный контракт взаимодействия (API).<br>1. Режим библиотеки: Функция `show_dialog()` возвращает `dict` или `None`.<br>2. Режим subprocess: Модуль выводит результат в `stdout` в формате JSON: `{"status": "success", "data": {...}}`, `{"status": "cancelled"}` или `{"status": "error", "message": "..."}` (при критической ошибке, код возврата != 0) и завершает работу с кодом 0 (или != 0 при ошибке). Интеграция не требует изменения кода вызывающей системы. |

## 3. Нефункциональные требования

| ID | Версия | Приоритет | Вид | Описание |

| NF-2c | 2.8 | Must | Надёжность | Нет необработанных исключений при проблемах с файлом сохранения. |
| NF-3a | 2.8 | Must | Безопасность | Пароль не сохраняется на диск. |

| NF-6 | 2.8 | Must | Совместимость | Работа без установленного Oracle Client. |
| NF-7 | 2.8 | Must | Развёртывание | Отсутствие скрытых зависимостей. CustomTkinter является единственной допустимой внешней зависимостью и должен быть упакован вместе с приложением через PyInstaller. При сборке через PyInstaller необходимо использовать аргумент `--hidden-import customtkinter` и убедиться, что директория `customtkinter/assets` включена в сборку для корректного отображения тем. |
