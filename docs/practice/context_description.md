Контекст задачи:
В рамках разработки ПО AlphaMeterQC планируюется к разработке модуль экранных форм для вывода актуальной информации о своевременности поступления, полноты и корректности данных, собранных со счётчиков электрической энергии. Для фоормирования данных форм помимо информации из БД (Oracle) ПО АльфаЦЕНТР необходимо наличие дополнительной справочной информации, для хранения которой необходимо разработать базу данных (БД).
Ожидается, что БД будеть работать на сервере в локальной сети.

Сценарий использования:
Инженеры отдела системного сопровождения на своих АРМ запускают AlphaMeterQC, которое:
- для формирования и отображания экранных форм комбинирует данные получаемые из БД АльфаЦЕНТР и разрабатываемой БД
- предоставляет пользовательский интерфейс для добавления, актуализации и удаления справочной информации.

Под справочной информацией подразумевается: атрибуты точек поставки/учёта, включающие  параметры, описывающие точки учёта с организационно-коммерческой стороны и технико-метрологические параметры.

Общее описание справочной информации:
- Учёт электрической энергии, потребляемой именованными Потребителями, осуществляется > 0 именованными точками учёта (далее - ТУ), которые в пределах объекта/электоустановки (далее - Объект) идентифицируется уникальным номером присоединения/фидера в БД АльфаЦЕНТР
- Каждая ТУ относятся к одному из именованных объектов, которые идентифицируется уникальным номером в БД АльфаЦЕНТР и имеют именованное место расположения, при это в одном месте расположения может быть > 0 объектов
- Именованные организации имеют > 0 именованных структурных подразделений, каждое из которых может иметь >= 0 контактных лиц из числа работников, характеризующихся должностью, фамилией, именем, отчеством, контактным телефоном и электронной почтой, и иметь в эксплуатационной ответственности > 0 объектов. Эти организации одновременно могут являтся Потребителем.
- По каждой ТУ может храниться информация о:
    - статусе (ОРЭМ, МТП, ДСД, ....), каждый из которых характеризуется максимально допустимым временем отсутствия поступления данных в БД АльфаЦЕНТР
    - заявке или заявках, каждая из которых характеризуется номером, датой начала и окончания
    - текстовых примечаниях
