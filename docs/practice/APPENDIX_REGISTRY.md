# Реестр соответствия приложений отчёта и артефактов

Настоящий реестр фиксирует соответствие между буквенными обозначениями приложений, используемыми в тексте отчёта по производственной практике (файл `docs/practice/report.txt`), и файлами артефактов, хранящимися в директории `docs/specs/db/`.

**Правила ведения реестра:**
1. При создании нового артефакта — сразу добавлять запись в реестр.
2. Буквы приложений — сквозная нумерация по алфавиту (А, Б, В, Г, Д, ...).
3. При удалении или переименовании артефакта — актуализировать запись.
4. Перед каждым новым разделом отчёта — сверяться с реестром для исключения дублирования букв.
