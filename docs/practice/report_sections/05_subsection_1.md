## 5.1 Настройка резервного копирования и восстановления данных

**Цель и задачи**

На этапе настройки резервного копирования решалась задача обеспечения сохранности данных справочной БД AlphaMeterQC в соответствии с требованиями технического задания (Приложение Д, раздел 3.5.2): возможность резервного копирования без остановки работы приложения, возможность восстановления из резервной копии, разработка регламента резервного копирования.

**Выбор стратегии резервного копирования**

На основе анализа требований к надёжности (доступность БД не менее 99,5 %, круглосуточный режим работы 24/7) и характера данных (справочная информация, изменяемая не чаще нескольких раз в день) была выбрана комбинированная стратегия резервного копирования:

| Тип копирования | Периодичность | Время выполнения | Срок хранения | Инструмент |
|:----------------|:--------------|:-----------------|:--------------|:-----------|
| Полная (custom format) | Ежедневно | 02:00 | 30 дней | pg_dump |
| Непрерывное архивирование WAL | Непрерывно | — | 7 дней | WAL archiving |

**Обоснование выбора:**
а) **Custom format** (`pg_dump --format=custom`) обеспечивает сжатие данных, возможность выборочного восстановления отдельных таблиц и параллельное выполнение;
б) **Ежедневное полное копирование** достаточно для справочных данных, которые изменяются нечасто;
в) **Срок хранения 30 дней** позволяет восстановить данные при обнаружении ошибки в течение месяца;
г) **Непрерывное архивирование WAL** обеспечивает восстановление на любой момент времени (point-in-time recovery) в пределах 7 дней.

**Разработка скриптов резервного копирования и восстановления**

Были разработаны два bash-скрипта (Приложения С и Т):

**Скрипт резервного копирования** (`16_backup_script.sh`, Приложение С) выполняет:
а) проверку доступности PostgreSQL перед началом копирования;
б) полное резервное копирование в custom-формате с помощью `pg_dump`;
в) ведение журнала операций с временными метками;
г) автоматическую ротацию копий (удаление копий старше 30 дней).

**Скрипт восстановления** (`17_restore_script.sh`, Приложение Т) выполняет:
а) проверку наличия и доступности файла резервной копии;
б) завершение всех активных подключений к восстанавливаемой БД;
в) восстановление с использованием `pg_restore` с параметрами `--clean` и `--if-exists` (пересоздание объектов);
г) ведение журнала операций.

**Настройка автоматического резервного копирования**

Для автоматизации резервного копирования настроено выполнение скрипта через планировщик cron:

```bash
# Ежедневное резервное копирование в 02:00
0 2 * * * /usr/local/bin/backup_alphameterqc.sh >> /var/log/alphameterqc_backup.log 2>&1
```

Время выполнения (02:00) выбрано как период минимальной активности пользователей, что исключает влияние резервного копирования на работу ПО AlphaMeterQC.

**Процедура восстановления**

Разработана и документирована процедура восстановления данных:

1. Остановить ПО AlphaMeterQC (завершить подключения к БД).
2. Выполнить скрипт восстановления с указанием файла резервной копии.
3. Проверить целостность восстановленных данных (количество таблиц, количество записей).
4. Запустить ПО AlphaMeterQC.
5. Проверить корректность формирования экранных форм.
