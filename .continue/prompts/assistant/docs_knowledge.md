## Структура документации
| Путь | Назначение |
|------|------------|
| `docs/specs/<module>/` | Постоянные спецификации модуля (концепция, требования, use cases, SRS, RTM, доменная модель, UI-спецификация) |
| `docs/adr/` | Архитектурные решения (ADR) |
| `docs/api/` | Описание API-контрактов (если применимо) |
| `docs/templates/` | Общие шаблоны (workflow, stage-output, code-review, feature-spec) |
| `docs/temp/` | Временные артефакты, общие для проекта |
| `docs/temp/<module>/` | Временные артефакты, относящиеся к конкретному модулю |

## Правила именования артефактов
- **Файлы спецификаций:** `<номер>-<тип>-<module>.md`  
  Пример: `01-concept-login.md`, `04-srs-login.md`
- **ADR:** `adr-<номер>-<краткое-название>.md`  
  Пример: `adr-001-use-customtkinter.md`
- **Временные артефакты (stage-output):** `<module>-<этап>-<дата>.md`  
  Пример: `login-spec-arch-2025-03-21.md`
- **ID требований/UC:** следуй нумерации из шаблонов (например, `UC.LOGIN.D1.01`, ``SR-1`, `F-01`)

## Управление версиями спецификаций
- Версионирование семантическое: `MAJOR.MINOR`
  - **MAJOR** — несовместимые изменения (удаление/изменение требований, архитектурные сдвиги).
  - **MINOR** — добавление требований, уточнения, исправления без нарушения обратной совместимости.
- При каждом изменении спецификации:
  - Увеличивается версия.
  - Обновляется `date`.
  - В конец файла или в отдельный `CHANGELOG.md` (в той же папке) добавляется запись об изменении.
- Пример записи в CHANGELOG:

  | Версия | Дата       | Тип      | Описание                                      |
  |--------|------------|----------|-----------------------------------------------|
  | 1.1    | 2025-03-22 | Added    | Требование F-05: поддержка OAuth2             |
  | 1.1    | 2025-03-22 | Changed  | UC.LOGIN.D1.01: уточнён формат токена         |

- Версия спецификации должна быть согласована с версией соответствующего ADR: если ADR меняется — спецификация тоже получает новую версию (номер может отличаться, но дата обновления должна совпадать).

## Шаблоны
Шаблоны в `docs/templates/`:
- не редактировать напрямую, а в случае необходимости предлагать корректировки;
- артефакты, созданные на основе шаблонов, разрешено адаптировать под конкретный модуль (задачу), сохраняя обязательные поля и структуру
