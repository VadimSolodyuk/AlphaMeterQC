## ПРАВИЛО 1. `edit_existing_file` — ЗАПРЕЩЁН НАВСЕГДА

**Никогда не используй `edit_existing_file`.** Этот инструмент **полностью перезаписывает файл** содержимым параметра `changes`. Он НЕ делает частичное редактирование — он заменяет ВЕСЬ файл целиком. Передача только изменяемых строк (даже с `// ... existing code ...`) **гарантированно уничтожает остальное содержимое файла**.

**Единственное исключение**: создание нового файла через `create_new_file`.

---

## ПРАВИЛО 2. ТОЛЬКО `single_find_and_replace` для изменений

Для всех изменений в существующих файлах используй **исключительно `single_find_and_replace`**. Он делает точную замену указанной строки на новую.

**Обязательные требования:**
- Всегда указывай **3-5 строк контекста** вокруг изменяемого места для обеспечения уникальности совпадения
- Скопируй точный текст из файла (включая пробелы и отступы) с помощью `read_file` перед редактированием
- Если замена не уникальна — расширь контекст, а не рискуй

---

## ПРАВИЛО 3. НИКОГДА не вызывай edit-инструменты параллельно

`single_find_and_replace` может вызываться **только последовательно, по одному разу за раз**. Запрещено вызывать несколько edit-инструментов одновременно (в одном блоке вызовов).

---

## ПРАВИЛО 4. Перед каждым edit — перечитай файл

**Всегда вызывай `read_file` непосредственно перед каждым edit**, даже если только что читал этот файл. Это необходимо, чтобы:
- Избежать race condition с изменениями пользователя
- Получить актуальное содержимое файла
- Точно скопировать контекст для замены

---

## ПРАВИЛО 5. Не более 3 файлов за раунд

За один раунд работы можно редактировать **не более 3 файлов**. Если требуется изменить больше — разбей на несколько раундов, дожидаясь подтверждения пользователя.

---

## ПРАВИЛО 6. После каждого edit — проверяй diff

Сразу после каждого вызова `single_find_and_replace` вызывай `view_diff`, чтобы убедиться, что изменения внесены корректно и не повреждены соседние участки.

---

## ПРАВИЛО 7. При ошибке редактирования

Если `single_find_and_replace` завершился ошибкой:
1. **Перечитай файл** через `read_file` (содержимое могло измениться)
2. **Скопируй более точный контекст** (не менее 5 строк)
3. **Попробуй снова** с уточнённым контекстом

---

## ПРАВИЛО 8. Для создания новых файлов — только `create_new_file`

Новые файлы создавай исключительно через `create_new_file`. Не используй для этого `edit_existing_file` или другие инструменты.
