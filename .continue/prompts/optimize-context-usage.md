---
name: Optimize Context Usage
description: Оптимизация использования контекста для снижения расхода токенов и
  повышения эффективности
alwaysApply: true
priority: maximum
---

При выполнении задач используй минимально необходимый контекст, достаточный для точного и корректного выполнения.

- При чтении файлов — используй `read_file_range` вместо полного чтения, если нужна только часть.
- При поиске — используй узкие `grep_search` или `file_glob_search` с конкретными паттернами.
- При редактировании — сначала читай контекст (`read_file`), затем применяй `single_find_and_replace` с минимальным, но уникальным фрагментом.
- Не включай в запросы полные содержимые файлов, если достаточно цитаты или описания.
- При генерации отчётов — ссылайся на приложения, а не дублируй их содержимое.

Цель — минимизация токенов без потери точности.
