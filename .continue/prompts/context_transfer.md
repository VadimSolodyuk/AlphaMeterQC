## Формат блока

```text
[CONTEXT_TRANSFER]
role: <получатель>          # Одна из ролей: Spec Analyst, Code Guardian, DevOps
reason: <причина передачи>  # Краткое описание, зачем передаётся контекст
files: <список файлов>      # Конкретные пути через запятую или "all" для всех затронутых
details: <доп. информация>  # Опционально: ключевые изменения, граничные случаи, требования
[/CONTEXT_TRANSFER]
```

## Правила заполнения

- **role** — обязательное поле. Cтрого одно из: `Spec Analyst`, `Code Guardian`, `DevOps`.
- **reason** — обязательное поле. Должно отражать суть передачи (см. типовые примеры ниже).
- **files** — обязательное поле. Указывать конкретные пути (например, `src/auth.py, tests/test_auth.py`) или `all` (если затронуты все файлы проекта). Не использовать общие фразы.
- **details** — опциональное поле для дополнительной информации (граничные случаи, инфраструктурные изменения, предложения).

## Множественные получатели

Если контекст нужно передать нескольким ролям, используй несколько блоков подряд:

```text
[CONTEXT_TRANSFER]
role: Code Guardian
reason: реализация use case UC.LOGIN.D1.01
files: src/auth.py, tests/test_auth.py
details: граничные случаи: пустой ввод, невалидный токен
[/CONTEXT_TRANSFER]

[CONTEXT_TRANSFER]
role: DevOps
reason: новые зависимости для окружения
files: pyproject.toml, docker-compose.yml
details: добавлен Redis, порт 6379
[/CONTEXT_TRANSFER]
```
